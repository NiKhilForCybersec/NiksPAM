Cloud KMS Services

Major cloud providers offer managed KMS services that integrate with their ecosystems. This page compares AWS KMS, Azure Key Vault, and GCP Cloud KMS, covering features, security models, and best practices.

Cloud KMS Overview

Cloud KMS services provide managed key management without the complexity of operating your own infrastructure. They integrate natively with cloud services and provide scalable, highly available cryptographic operations.

Why Use Cloud KMS?

Native Integration

Seamless encryption for cloud storage, databases, compute, and other services with minimal configuration.

Managed Service

No infrastructure to operate—the cloud provider handles availability, scaling, and maintenance.

HSM-Backed Options

FIPS 140-2 Level 3 certified HSM backing available for compliance requirements.

Pay-Per-Use

No upfront hardware costs—pay only for keys stored and operations performed.

AWS Key Management Service (KMS)

AWS KMS is AWS's managed encryption key service, integrated with over 100 AWS services for seamless data encryption.

Key Types

Key Type Description Use Case
AWS Managed Keys Created and managed by AWS for specific services (aws/s3, aws/ebs) Simplest option for service encryption
Customer Managed Keys (CMK) You create and control, full policy management Most control, cross-account access
Customer Managed Keys with Imported Material You import your own key material Regulatory requirements, hybrid scenarios
Custom Key Stores (CloudHSM) Keys stored in your CloudHSM cluster Highest security, FIPS 140-2 Level 3

Key Features

  • Envelope encryption: Generate data keys for client-side encryption
  • Automatic rotation: Annual rotation for customer managed keys
  • Key policies: Fine-grained access control per key
  • Multi-region keys: Replicate keys across regions for DR
  • CloudTrail integration: Complete audit trail of key usage
  • Grants: Temporary, delegated access to keys

AWS CloudHSM

For workloads requiring dedicated HSMs, AWS CloudHSM provides single-tenant FIPS 140-2 Level 3 HSMs in your VPC.

CloudHSM Characteristics
  • Full control over HSM—you manage users and keys
  • PKCS#11, JCE, OpenSSL, and Microsoft CNG interfaces
  • Cluster mode for high availability
  • Can integrate with KMS as custom key store

Pricing Model

  • KMS keys: $1/month per customer managed key
  • API requests: $0.03 per 10,000 requests
  • CloudHSM: ~$1.50/hour per HSM instance

Azure Key Vault

Azure Key Vault provides secure storage for keys, secrets, and certificates with integration across Azure services.

Key Vault Tiers

Tier Key Protection FIPS Level Use Case
Standard Software-protected FIPS 140-2 Level 1 Development, non-sensitive workloads
Premium HSM-protected FIPS 140-2 Level 2 Production, compliance needs
Managed HSM Dedicated HSM pool FIPS 140-2 Level 3 Highest security requirements

Object Types

  • Keys: Cryptographic keys for encryption and signing
  • Secrets: Arbitrary secret strings (passwords, connection strings, API keys)
  • Certificates: X.509 certificates with lifecycle management

Key Features

  • Soft delete: Recover accidentally deleted keys (retention period)
  • Purge protection: Prevent permanent deletion during retention period
  • Bring Your Own Key (BYOK): Import keys from on-premises HSMs
  • Azure AD integration: RBAC and managed identities for access
  • Private endpoint: Access via private network connection
  • Key rotation policy: Automatic rotation on schedule

Azure Dedicated HSM

For single-tenant HSM requirements, Azure Dedicated HSM provides Thales Luna HSMs in Azure datacenters.

Pricing Model

  • Standard: $0.03 per 10,000 operations
  • Premium: $1/month per key + $0.03 per 10,000 operations
  • Managed HSM: ~$3/hour per HSM pool

Google Cloud KMS

Google Cloud KMS provides cryptographic key management integrated with GCP services and workloads.

Protection Levels

Protection Level Description Use Case
SOFTWARE Software-based key protection General workloads
HSM FIPS 140-2 Level 3 HSM Compliance requirements
EXTERNAL Key managed externally (Cloud EKM) Keys never in GCP
EXTERNAL_VPC External via VPC connection Private connectivity to external KMS

Key Features

  • Key hierarchy: Key rings contain keys, keys contain versions
  • Automatic rotation: Configurable rotation period
  • Asymmetric keys: RSA and EC for signing and encryption
  • Cloud EKM: Use keys from external key managers (Thales, Fortanix)
  • VPC Service Controls: Network isolation for key operations
  • IAM integration: Role-based access control

Cloud External Key Manager (Cloud EKM)

Cloud EKM allows you to use keys managed in supported external key management partners. The key material never enters GCP.

Hold Your Own Key (HYOK)

Cloud EKM enables "Hold Your Own Key" scenarios where you maintain key material outside GCP. If you revoke the external key, GCP services lose access to encrypted data—giving you ultimate control.

Pricing Model

  • Active key versions: $0.06/month per key version
  • Key operations: $0.03 per 10,000 operations
  • HSM keys: $1-2.50/month per key version + operations

Multi-Cloud Strategies

Organizations using multiple clouds face key management challenges. Here are common approaches:

Option 1: Cloud-Native Per Cloud

Use Each Cloud's Native KMS
  • Approach: AWS KMS for AWS, Azure Key Vault for Azure, GCP KMS for GCP
  • Pros: Best integration, simplest to implement
  • Cons: Multiple key management systems, no unified view
  • Best for: Loosely coupled multi-cloud, cloud-specific workloads

Option 2: External KMS

Third-Party KMS Across Clouds
  • Approach: Thales CipherTrust, Fortanix, or HashiCorp Vault across all clouds
  • Pros: Unified key management, consistent policies
  • Cons: Additional complexity, may lose native integration benefits
  • Best for: Strong multi-cloud strategy, regulatory requirements

Option 3: Hybrid

External KMS for Master Keys, Cloud KMS for DEKs
  • Approach: Store master keys in external KMS (or GCP EKM), let cloud KMS handle data keys
  • Pros: Central control of master keys, cloud-native performance for operations
  • Cons: Complex architecture
  • Best for: Hold-Your-Own-Key requirements, high-security scenarios

Best Practices

Key Management

Key Management Best Practices
  • Use customer managed keys: Don't rely solely on provider-managed keys for sensitive data
  • Enable automatic rotation: Configure appropriate rotation periods
  • Separate keys by purpose: Different keys for different applications/environments
  • Use key aliases: Reference keys by alias, not key ID, for easier rotation
  • Document key inventory: Maintain records of all keys and their purposes

Access Control

Access Control Best Practices
  • Least privilege: Grant minimum necessary permissions to each identity
  • Use managed identities: Prefer managed identities over long-lived credentials
  • Separate admin from usage: Different roles for key administration vs. cryptographic operations
  • Review access regularly: Audit who has access to sensitive keys
  • Use conditions: Add IP restrictions, time-based access where appropriate

Monitoring & Audit

Monitoring Best Practices
  • Enable logging: CloudTrail, Azure Monitor, Cloud Audit Logs
  • Alert on sensitive operations: Key deletion, policy changes, unusual usage
  • Review logs regularly: Look for unauthorized access attempts
  • Integrate with SIEM: Centralize key management logs with other security events

Disaster Recovery

DR Best Practices
  • Multi-region keys: Use multi-region keys (AWS) or replicate key material
  • Backup procedures: Document how to recover if keys are lost
  • Test recovery: Regularly test key recovery procedures
  • Avoid single points of failure: Don't depend on a single key for critical operations

Key Takeaways

Cloud KMS services (AWS KMS, Azure Key Vault, GCP Cloud KMS) provide managed key management with native cloud integration. All offer HSM-backed options for compliance. Choose based on your cloud strategy—cloud-native for single cloud, external KMS for multi-cloud. Follow best practices for key management, access control, and monitoring.
