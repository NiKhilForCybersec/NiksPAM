Cryptographic Basics

Before diving deep into KMS, you need to understand the cryptographic concepts it manages. This page covers encryption, hashing, digital signatures, and key hierarchies in plain language.

Why Cryptography Matters

Cryptography provides the mathematical foundation for three critical security properties:

Confidentiality

Only authorized parties can read the data. Encryption transforms readable data into unreadable ciphertext.

Integrity

Data hasn't been tampered with. Hashing and signatures detect any modifications to the original data.

Authenticity

Data came from who it claims to be from. Digital signatures prove the origin of data.

You Don't Need to Be a Mathematician

Working with KMS doesn't require understanding the math behind cryptographic algorithms. You need to understand what each type of cryptography does, when to use it, and how to manage the keys. That's what this page covers.

Common Symmetric Algorithms

  • AES (Advanced Encryption Standard): The gold standard. AES-256 is used for classified information by the US government.
  • ChaCha20: Modern alternative to AES, faster in software implementations without hardware acceleration.
  • 3DES (Triple DES): Legacy algorithm, being phased out. Don't use for new implementations.

The Key Distribution Problem

Symmetric encryption's biggest challenge: both parties need the same secret key. If you're sending encrypted data to someone, how do you securely send them the key? You can't encrypt it (they don't have the key yet!). This is where asymmetric encryption helps.

Asymmetric Encryption

Asymmetric encryption (also called public-key cryptography) uses a mathematically linked pair of keys: a public key and a private key. What one key encrypts, only the other can decrypt.

How It Works: Encryption

When someone wants to send you encrypted data:

1

Sender Gets Your Public Key

You share your public key openly. It's not a secret‚Äîanyone can have it.

2

Sender Encrypts with Your Public Key

The sender encrypts their message using your public key. Once encrypted, even the sender can't decrypt it.

3

You Decrypt with Your Private Key

Only your private key can decrypt the message. Since only you have the private key, only you can read it.

Key Characteristics

Aspect Description
Speed Much slower than symmetric‚Äîtypically 100-1000x slower
Key Size RSA: 2048-4096 bits; ECC: 256-384 bits (smaller but equivalent security)
Use Cases Key exchange, digital signatures, TLS handshake, email encryption (S/MIME, PGP)
Advantage No key distribution problem‚Äîyou can publish your public key

Common Asymmetric Algorithms

  • RSA: The most widely used. RSA-2048 is the minimum; RSA-4096 for long-term security.
  • ECC (Elliptic Curve Cryptography): Smaller keys, same security. ECC-256 ‚âà RSA-3072 in strength.
  • Ed25519: Modern elliptic curve algorithm, excellent for signatures (used in SSH keys).

Hybrid Encryption: Best of Both Worlds

In practice, we combine both types. Asymmetric encryption securely exchanges a symmetric key, then symmetric encryption handles the bulk data. This is how TLS works: RSA/ECC for the handshake, AES for the actual data.

Hashing

A hash function takes any input and produces a fixed-size output (the "hash" or "digest"). Unlike encryption, hashing is one-way‚Äîyou cannot recover the original data from the hash.

Key Properties of Hash Functions

Deterministic

Same input always produces the same hash. "Hello" will always hash to the same value.

One-Way

Cannot reverse the hash to get the original input. No "un-hash" function exists.

Avalanche Effect

Tiny input change = completely different hash. "Hello" and "hello" have unrelated hashes.

Collision Resistant

Extremely hard to find two different inputs that produce the same hash.

Common Uses of Hashing

Use Case How Hashing Helps
Password Storage Store hash of password, not the password itself. Compare hashes during login.
File Integrity Hash of a file proves it hasn't been modified. Download sites show SHA-256 checksums.
Digital Signatures Sign the hash of a document instead of the entire document (much faster).
Message Authentication HMAC combines hashing with a key to verify both integrity and authenticity.
Deduplication Same hash = same content. Storage systems use this to avoid storing duplicates.

Common Hash Algorithms

  • SHA-256: Part of SHA-2 family. 256-bit output. The current standard.
  • SHA-384/SHA-512: Larger outputs for extra security margin.
  • SHA-3: Newest standard, different design than SHA-2. Good for defense in depth.
  • MD5: BROKEN ‚Äî Never use for security. Only for non-security checksums.
  • SHA-1: DEPRECATED ‚Äî Collision attacks exist. Avoid.

Digital Signatures

A digital signature proves that a message came from a specific sender and hasn't been modified. It's like a handwritten signature but mathematically verifiable and impossible to forge.

How Digital Signatures Work

1

Hash the Document

Create a hash (digest) of the document you want to sign. This creates a fixed-size fingerprint.

2

Encrypt Hash with Private Key

The signer encrypts the hash using their private key. This encrypted hash IS the digital signature.

3

Send Document + Signature

The original document and the signature are sent together. The document is NOT encrypted (unless you want it to be).

4

Recipient Verifies

Recipient decrypts the signature using the sender's public key (revealing the original hash), then hashes the received document and compares. If they match, the signature is valid.

What Digital Signatures Prove

  • Authentication: The signature could only be created by someone with the private key
  • Integrity: Any modification to the document would produce a different hash, failing verification
  • Non-repudiation: The signer cannot deny signing‚Äîonly they have the private key

Common Uses

  • Code Signing: Software publishers sign their code so you know it's authentic
  • TLS Certificates: Certificate authorities sign certificates to vouch for website identity
  • Email (S/MIME): Sign emails to prove they came from you
  • Documents: Legally sign contracts and documents electronically
  • Software Updates: Verify updates came from the vendor, not an attacker

Why Hierarchies Matter

Benefits of Key Hierarchies
  • Master Key Protection: The master key rarely (or never) leaves the HSM, dramatically reducing exposure
  • Efficient Rotation: To rotate a KEK, just re-wrap the DEKs with the new KEK‚Äîno need to re-encrypt all data
  • Separation of Duties: Different teams can manage different levels of the hierarchy
  • Compliance: Many standards (PCI-DSS, etc.) require key-encrypting keys to be separate from data keys
  • Performance: DEKs can be cached closer to applications since they're protected by the hierarchy

Putting It All Together

Let's see how these concepts combine in a real-world scenario: secure communication using TLS (HTTPS).

What Happens When You Visit https://bank.com

1

Server Sends Certificate

The server sends its TLS certificate, which contains its public key and is digitally signed by a trusted Certificate Authority.

2

Browser Verifies Signature

Your browser uses the CA's public key to verify the certificate's digital signature. This proves the certificate is authentic.

3

Key Exchange (Asymmetric)

Browser and server perform an asymmetric key exchange (using ECDH or RSA) to agree on a shared secret. Only they know this secret.

4

Derive Session Keys (Symmetric)

Both sides derive symmetric encryption keys from the shared secret. These are used for the actual data.

5

Encrypted Communication

All traffic is now encrypted with AES (symmetric), with integrity verified using HMAC (hashing with a key).

This single connection uses asymmetric cryptography, symmetric cryptography, digital signatures, and hashing‚Äîall working together.

Key Takeaways

You now understand the cryptographic building blocks that KMS manages: symmetric encryption for bulk data, asymmetric encryption for key exchange and signatures, hashing for integrity, digital signatures for authenticity, and key hierarchies for security and manageability. This foundation will help you understand how KMS protects your organization's data.
