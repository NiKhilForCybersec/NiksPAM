PAM & KMS Glossary

A comprehensive reference of terminology used in Privileged Access Management, Key Management Systems, and related security domains. Use this as a quick reference during your studies or work.

A

A2A (Application-to-Application)

Password management for non-human accounts where applications need to authenticate to other applications or databases. PAM solutions provide APIs for applications to retrieve credentials without hard-coding them.

Access Control List (ACL)

A list that defines which users or systems have permission to access specific resources and what operations they can perform.

AES (Advanced Encryption Standard)

A symmetric encryption algorithm adopted by the U.S. government as the standard for encrypting sensitive data. AES-256 (256-bit key) is the most common strength used in enterprise environments.

Asymmetric Encryption

Encryption using a mathematically related key pair: a public key (can be shared) and a private key (must be kept secret). Also called public-key cryptography. Used for digital signatures, key exchange, and encrypting small amounts of data.

Audit Trail

A chronological record of system activities that enables reconstruction and examination of events. In PAM, this includes who accessed what credential, when, from where, and what they did during their session.

B

Break-Glass Access

Emergency access procedure that bypasses normal approval workflows when immediate access is required. Typically triggers enhanced logging and alerts, and requires post-incident justification.

Brokered Access

Access method where a proxy (the broker) establishes the connection to the target system on behalf of the user. The user never directly connects to the target or sees the credential. This is how PAM session management works.

C

CA (Certificate Authority)

An entity that issues digital certificates. The CA verifies the identity of certificate requesters and digitally signs certificates with its private key. CAs can be public (DigiCert, Let's Encrypt) or private (internal enterprise CA).

Checkout/Checkin

The process of temporarily claiming exclusive access to a credential (checkout) and returning it when finished (checkin). Prevents multiple people from using the same credential simultaneously.

Ciphertext

The encrypted form of data. Ciphertext is unreadable without the decryption key. Contrast with plaintext.

CPM (Central Password Manager)

CyberArk-specific term for the component that automatically rotates passwords on target systems. Generic term: Password Rotation Engine.

Credential Injection

The process where PAM automatically enters credentials into the login prompt of a target system so users never see or type the password. This is done by the session manager component.

Credential Vaulting

Securely storing credentials in an encrypted repository (vault) with access controls, audit logging, and automated management capabilities.

Cryptoperiod

The time span during which a specific cryptographic key is authorized for use. After the cryptoperiod expires, the key should be rotated.

D

DEK (Data Encryption Key)

A key used directly to encrypt actual data (files, database records, etc.). DEKs are typically protected by higher-level Key Encryption Keys (KEKs) in a key hierarchy.

Digital Signature

A cryptographic mechanism that proves the authenticity and integrity of a message or document. Created using the signer's private key and verified using their public key.

Discovery

The process of scanning an environment to identify privileged accounts that should be managed by PAM. Can be network-based, directory-based, or cloud-based.

Dual Control

A security principle requiring two or more people to complete a sensitive action. In PAM, this might mean two approvers must approve a request, or two people must be present for break-glass access.

Dynamic Secrets

Credentials generated on-demand with short lifespans that are automatically revoked after use. A key feature of HashiCorp Vault and modern secrets management. Eliminates standing credentials.

E

Envelope Encryption

A key management pattern where data is encrypted with a Data Encryption Key (DEK), and the DEK is encrypted with a Key Encryption Key (KEK). This allows efficient key rotation without re-encrypting all data.

Ephemeral Credentials

Short-lived credentials that exist only for the duration of a specific session or task. After use, they're automatically revoked or expire. Reduces the window of exposure if credentials are compromised.

F

FIPS 140-2/140-3

Federal Information Processing Standards that specify security requirements for cryptographic modules. FIPS 140-2 Level 3 is commonly required for HSMs in regulated environments. FIPS 140-3 is the newer version.

H

Hash

A fixed-size output produced by a hash function from variable-size input. Hashes are one-way (cannot be reversed) and deterministic (same input always produces same output). Used for integrity verification and password storage.

HMAC (Hash-based Message Authentication Code)

A type of message authentication code calculated using a cryptographic hash function and a secret key. Provides both integrity and authenticity.

HSM (Hardware Security Module)

A dedicated physical device that securely generates, stores, and manages cryptographic keys. HSMs are tamper-resistant and provide the highest level of key protection. Required for many compliance standards.

I

IdP (Identity Provider)

A system that creates, maintains, and manages identity information and provides authentication services. Examples: Azure AD, Okta, Ping Identity. PAM solutions typically integrate with IdPs for SSO.

J

JIT (Just-In-Time) Access

A PAM approach where privileged access is granted only when needed and for only as long as needed. Users don't have standing privileges; they request access which is granted temporarily.

K

KEK (Key Encryption Key)

A key used to encrypt other keys rather than data directly. Part of a key hierarchy. Master keys and key-wrapping keys are types of KEKs.

Key Hierarchy

A structured arrangement of cryptographic keys where higher-level keys protect lower-level keys. Typically: Master Key → Key Encryption Keys → Data Encryption Keys → Data.

Key Rotation

The process of replacing a cryptographic key with a new one. For symmetric keys, this may involve re-encrypting data. For asymmetric keys, this involves generating new key pairs.

Key Wrapping

Encrypting one key with another key for secure storage or transmission. The protected key is "wrapped" and can only be used after being "unwrapped" with the wrapping key.

KMIP (Key Management Interoperability Protocol)

An OASIS standard that defines a single, comprehensive protocol for communication between key management systems and encryption systems. Enables interoperability between different vendor solutions.

KMS (Key Management System)

A system that manages the lifecycle of cryptographic keys: generation, storage, distribution, rotation, and destruction. Can be software, hardware (HSM), or cloud-based.

L

Least Privilege

The security principle that users should have only the minimum access necessary to perform their job functions. A core principle driving PAM and JIT access implementations.

Live Session Monitoring

The ability to watch privileged sessions in real-time. Security teams can observe what users are doing and terminate sessions if suspicious activity is detected.

M

Master Key

The top-level key in a key hierarchy. Protects all other keys. Master keys are typically stored in HSMs and never exposed in plaintext outside the HSM.

MFA (Multi-Factor Authentication)

Authentication requiring two or more verification factors: something you know (password), something you have (token, phone), or something you are (biometric). Critical for PAM access.

N

NLA (Network Level Authentication)

A security feature for RDP that requires authentication before a full remote desktop connection is established. PAM session managers must support NLA for Windows systems.

O

OIDC (OpenID Connect)

An authentication layer on top of OAuth 2.0. Commonly used for SSO integration between PAM solutions and identity providers.

Orphan Account

A privileged account that no longer has an associated owner (e.g., the employee left the company). Orphan accounts are security risks and should be identified and remediated.

P

PAM (Privileged Access Management)

A category of solutions that secure, control, and monitor privileged access to critical systems and data. Includes credential vaulting, session management, and access control.

Password Rotation

Automatically changing passwords on a schedule or after use. PAM solutions connect to target systems to change passwords and store the new values in the vault.

PIM (Privileged Identity Management)

Managing the identities that have or can obtain privileged access. Often used interchangeably with PAM, though PIM focuses more on the identity lifecycle aspect.

PKI (Public Key Infrastructure)

The framework of policies, procedures, and technology for managing digital certificates and public/private key pairs. Includes Certificate Authorities, registration authorities, and certificate repositories.

Plaintext

Data in its original, readable form before encryption. Contrast with ciphertext.

Privileged Account

An account with elevated permissions that can perform administrative functions, access sensitive data, or make system-wide changes. Examples: root, Administrator, DBA accounts.

PSM (Privileged Session Manager)

The PAM component that acts as a proxy for privileged sessions. Enables credential injection, session recording, and live monitoring without users directly connecting to targets.

PTA (Privileged Threat Analytics)

Analytics capabilities that detect anomalous privileged behavior. Uses machine learning and behavioral analysis to identify potential threats or compromised accounts.

PVWA (Password Vault Web Access)

CyberArk-specific term for the web portal where users access the PAM system. Generic term: PAM Web Portal.

R

RBAC (Role-Based Access Control)

An access control method where permissions are assigned to roles, and users are assigned to roles. Simplifies access management compared to assigning permissions directly to users.

Reconciliation

The process of re-synchronizing credentials when PAM's stored password doesn't match the actual password on the target system. Uses a reconciliation account to reset the password.

RSA

A widely used asymmetric encryption algorithm named after its inventors (Rivest, Shamir, Adleman). Common for TLS certificates and code signing. Minimum recommended key size is 2048 bits.

S

Safe

CyberArk term for a logical container within the vault that holds a group of credentials with common access policies. Other vendors may call this a folder, container, or secret path.

SAML (Security Assertion Markup Language)

An XML-based standard for exchanging authentication and authorization data between parties, particularly between an identity provider and a service provider. Used for SSO.

Secret

Any sensitive credential or data that should be protected. Includes passwords, API keys, tokens, certificates, and encryption keys.

Secrets Management

The process of securely storing, accessing, and distributing secrets (credentials, API keys, etc.) to applications and services. HashiCorp Vault is a leading secrets management solution.

Service Account

An account used by an application, service, or automated process rather than a human user. Service accounts often have elevated privileges and don't rotate passwords frequently, making them high-value targets.

Session Recording

Capturing a complete record of a privileged session including video, keystrokes, and commands. Used for audit, compliance, and forensic investigation.

SSH Key

An asymmetric key pair used for SSH authentication. The private key stays with the user; the public key is placed on servers. PAM can vault and rotate SSH keys.

SSO (Single Sign-On)

Authentication scheme that allows users to log in once and access multiple applications without re-authenticating. PAM solutions integrate with SSO via SAML or OIDC.

Symmetric Encryption

Encryption using the same key for both encryption and decryption. Fast and efficient for bulk data encryption. AES is the most common symmetric algorithm.

T

TDE (Transparent Data Encryption)

Database feature that encrypts data at rest without application changes. The database encrypts data when writing to disk and decrypts when reading. Managed encryption keys are essential for TDE.

TLS (Transport Layer Security)

Cryptographic protocol that provides secure communication over networks. Successor to SSL. Uses both asymmetric (for key exchange) and symmetric (for data encryption) cryptography.

V

Vault

A secure repository for storing sensitive credentials and secrets. In PAM, the vault is the central component that stores privileged credentials encrypted at rest with strict access controls.

Verification

In PAM, the process of testing that a stored credential can successfully authenticate to its target system. Verifications are run after rotations to ensure the new password works.

Z

Zeroization

The process of erasing sensitive data (especially cryptographic keys) from a device so it cannot be recovered. HSMs perform zeroization when tamper events are detected.

Zero Standing Privileges (ZSP)

A security model where users have no persistent privileged access. All privileged access is granted just-in-time based on request and approval, and automatically revoked after use.

Zero Trust

A security model based on "never trust, always verify." Assumes no user or system should be trusted by default, regardless of network location. PAM is a key component of Zero Trust architectures.
